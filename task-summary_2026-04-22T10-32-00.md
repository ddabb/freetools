# 任务总结 - 2026-04-22 10:32

## 项目：freetools 数织(Nonogram)约束传播优化

### 用户场景
5格，提示[2,1]。用户先填pos2+标X-pos3，再填pos1。

**期望**：pos0自动标X（因为已填的两个格子的位置排除了pos0作为[2]起点的所有可能）

**原问题**：约束只在同行同列传播，无法利用列信息推导行

### 两个根因

**Bug 1：约束范围过窄**
- `_doOp` 只检查操作所在的行和列
- 但列的信息可以帮助判断行（例如列标X约束了某格不是block起点）
- 修复：改用 `applyConstraints` 全棋盘扫描

**Bug 2：无合法排列时返回空**
- `solveLine` 当 `valid.length === 0` 时返回 `mustFill:[] mustEmpty:[]`
- 但此时恰恰说明**所有未知格子都不是block的一部分**——应该全部标X
- 修复：返回所有 line 中为0的位置作为 mustEmpty

### 验证
```
场景：[2,1] + 填pos2 + 标X-pos3 + 填pos1
  line=[0,1,1,-1,0]
  genPlacements: 无任何placement能兼容（pos3标X排除了所有placement）
  修复前: {mustFill:[], mustEmpty:[]} ❌
  修复后: {mustFill:[], mustEmpty:[0,4]} ✅

场景：[1,1] + 填pos0/2
  line=[1,0,1,0,0]
  修复后: {mustFill:[0,2], mustEmpty:[1,3,4]} ✅
```
