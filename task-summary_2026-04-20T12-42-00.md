## 项目：freetools 数织(Nonogram)约束传播修复

### Bug 描述
5格+提示[1,1]，用户填了两个位置（中间有间隔），算法应该自动把中间格子标记X，反而自动填了格子。

### 根因（两个叠加 bug）

**Bug 1：mustEmpty 逻辑错误**
- `genPlacements` 生成 placement 数组，语义：1=填，-1=标X，**0=组间gap（不确定）**
- 原代码 `if (valid[j][i] !== -1) allEmpty = false` 把 gap(0) 当作"不是mustEmpty"，永远漏掉gap位置
- 修复：`if (valid[j][i] === 1) mustEmptyFlag = false`（只要有placement在该位置填了，就非空）

**Bug 2：X标记兼容性检查缺失**
- 原代码 `line[j] !== placements[i][j]` 对 line=-1（用户标X）接受 placement=1（填）
- 修复：显式检查 `line[j] === -1 && placements[i][j] !== -1 → invalid`

### 验证
```
Line: [1,0,1,0,0]
修复前 Result: mustFill=[0,2], mustEmpty=[4]  ❌
修复后 Result: mustFill=[0,2], mustEmpty=[1,3,4]  ✅
```
