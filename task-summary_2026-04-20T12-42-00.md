# 任务总结 - 2026-04-20 12:42

## 项目：freetools 数织(Nonogram)约束求解器修复

### Bug 描述
用户填了5格中的2个（提示 `[1,1]`，中间有间隔），约束传播错误地自动填充中间格子。

### 根因
`solveLine` 函数中，placement 兼容性检查逻辑错误：
```javascript
// 旧代码（bug）
if (line[j] !== 0 && line[j] !== placements[i][j])
```
当 `line[j] === -1`（用户已标记 X）时，条件判断不严格：
- `line[j] !== 0` → true（进入检查）
- 但 `-1 !== placements[i][j]` 对 placement[j]=1 或 -1 都可能为 true
- 导致 X 被当作"未知"处理，约束反向推导

### 修复
```javascript
// 新代码（正确）
if (line[j] === 1 && placements[i][j] !== 1) { ok = false; break; }  // 用户填了必须匹配
if (line[j] === -1 && placements[i][j] !== -1) { ok = false; break; } // 用户标X必须是空格
```
显式区分：未知(0)=任意，填(1)=必须1，标X(-1)=必须-1。
