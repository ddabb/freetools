# 知识库建设指南

> 记录 FreeTools 知识库（随身百科）的建设规范、构建流程和使用说明。

## 一、知识库架构

### 1.1 整体架构

```
PortableKnowledge/           ← 知识库主目录（Git 仓库）
├── source/                  ← Markdown 源文件（知识碎片）
│   ├── 引论/
│   │   ├── 系统集成概述.md
│   │   └── 项目管理概述.md
│   ├── 项目整合管理/
│   └── ...
├── know/                    ← 构建产物（JSON，CDN 分发）
│   ├── all.json            ← 全量数据
│   ├── index.json          ← 索引
│   └── category/
├── build.js                ← 构建脚本
└── README.md
```

### 1.2 数据流向

```
Markdown 源文件（source/）
        ↓  build.js 构建
JSON 产物（know/）
        ↓  GitHub 推送
jsDelivr CDN
        ↓  小程序请求
知识库页面展示
```

## 二、源文件规范

### 2.1 Front Matter 格式

每个 Markdown 文件必须包含 YAML front matter：

```markdown
---
title: "系统集成项目管理工程师"
description: "系统集成项目管理工程师考试备考知识库，包含项目立项、系统设计等核心知识点。"
category: "引论"
tags: ["软考", "项目管理", "系统集成"]
---
```

### 2.2 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| title | 是 | 知识点标题 | "项目范围管理" |
| description | 是 | 简要描述，不超过200字 | "项目范围管理包括..." |
| category | 是 | 一级分类（章节名） | "项目范围管理" |
| tags | 是 | 标签数组，2-5个 | ["范围", "WBS", "范围基准"] |

### 2.3 正文格式

```markdown
# 项目范围管理

## 1. 范围管理概述

项目范围管理包括以下过程...

## 2. 规划范围管理

### 2.1 输入
- 项目章程
- 项目管理计划

### 2.2 工具与技术
- 专家判断
- 数据分析

### 2.3 输出
- 范围管理计划
- 需求管理计划
```

**写作规范**：
- 使用标准 Markdown 语法
- 标题层级清晰（# → ## → ###）
- 重点内容可用 **加粗** 标注
- 列表优先用有序列表
- 代码块用反引号

## 三、标签体系

### 3.1 允许的标签

```
# 考试类型
软考 | PMP | 系统架构师 | 网络工程师

# 知识领域
项目整合 | 项目范围 | 项目时间 | 项目成本 | 项目质量
项目资源 | 项目沟通 | 项目风险 | 项目采购 | 项目干系人

# 技术领域
操作系统 | 数据库 | 计算机网络 | 软件工程 | 信息安全

# 通用标签
概念 | 公式 | 流程图 | 记忆口诀 | 易错点 | 高频考点
```

### 3.2 标签命名规则

- 使用中文词或词组
- 避免特殊字符（`/` `:` `#` 等）
- 同一个概念尽量统一标签名
- 单个知识最多 5 个标签

## 四、构建流程

### 4.1 安装依赖

```bash
cd f:\SelfJob\PortableKnowledge
npm install
```

### 4.2 运行构建

```bash
node build.js
```

构建脚本会：
1. 读取 `source/` 目录下所有 `.md` 文件
2. 解析 front matter 和正文内容
3. 生成 `know/` 目录下的 JSON 文件
4. 输出构建报告

### 4.3 构建产物说明

| 文件 | 说明 | 用途 |
|------|------|------|
| `know/all.json` | 全量知识数据 | CDN 备份 |
| `know/index.json` | 知识索引 | 按分类/标签快速检索 |
| `know/category/*.json` | 按分类拆分的文件 | 按需加载 |

## 五、小程序端接入

### 5.1 数据加载

```javascript
const CDN_BASE = 'https://cdn.jsdelivr.net/gh/ddabb/FreeToolsPuzzle@main/know';

function loadKnowledgeIndex() {
  return wx.request({
    url: `${CDN_BASE}/index.json`,
    timeout: 10000,
  });
}

function loadCategory(cat) {
  return wx.request({
    url: `${CDN_BASE}/category/${cat}.json`,
    timeout: 10000,
  });
}
```

### 5.2 缓存策略

| 数据类型 | 缓存时间 |
|----------|----------|
| 索引数据 | 7 天 |
| 分类数据 | 7 天 |
| 详情数据 | 按需加载，不缓存 |

## 六、内容更新流程

### 6.1 新增知识

1. 在对应章节目录下创建 `.md` 文件
2. 填写 front matter
3. 编写正文内容
4. 运行 `node build.js`
5. Git push 触发 CDN 更新

### 6.2 修改知识

1. 找到对应 `.md` 文件
2. 修改 front matter 或正文
3. 运行 `node build.js`
4. Git push

### 6.3 删除知识

1. 删除对应 `.md` 文件
2. 运行 `node build.js`
3. Git push

## 七、质量检查

### 7.1 自检清单

> 以下条目为编写规范，实际编写时逐项确认：

- [x] front matter 字段完整
- [x] category 属于已定义分类
- [x] tags 符合标签体系
- [x] 正文无乱码
- [x] 构建无报错
- [x] 小程序内加载正常

### 7.2 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| build.js 报错 | front matter 格式错误 | 检查 YAML 语法 |
| 标签含特殊字符 | 用了 `/` 或 `#` | 替换为中文词 |
| 分类找不到 | category 与目录不匹配 | 核对章节名 |
| 文件名乱码 | 路径含中文 | 确保文件保存为 UTF-8 |
