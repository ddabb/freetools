# 小程序字体缺失解决方案

## 问题分析

根据项目中的问题描述，小程序存在以下字体相关问题：

1. **页面字体显示异常**：由于手机字体缺失，导致整个系统的页面像Word风格
2. **Canvas绘图字体效果缺失**：使用Canvas生成图片时，字体效果无法正常显示

## 核心原因

根据调研，微信小程序字体缺失的主要原因有：

1. **沙箱环境限制**：小程序运行在微信的隔离环境中，无法直接调用系统私有字体
2. **系统字体差异**：
   - iOS和Android系统字体库不同
   - 不同厂商的Android系统字体也有差异
3. **字体加载机制**：小程序的字体加载机制与传统Web不同

## 最佳解决方案

### 方案1：使用微信小程序安全字体（推荐）

这是最简单、最可靠的解决方案，优先使用微信官方推荐的安全字体。

#### 安全字体列表

**iOS系统**：
- PingFang SC (苹方) - 中文
- San Francisco - 英文/数字

**Android系统**：
- 思源黑体 - 中文
- Roboto - 英文/数字

**通用字体栈**：
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
```

### 方案2：使用wx.loadFontFace动态加载字体

如果确实需要使用特殊字体，可以使用微信提供的API动态加载字体。

**注意事项**：
- 字体文件必须托管在HTTPS服务器上
- 需要配置downloadFile合法域名
- Canvas组件需要使用Canvas 2D API才能支持

**使用示例**：
```javascript
wx.loadFontFace({
  family: 'CustomFont',
  source: 'url("https://your-domain.com/fonts/custom-font.ttf")',
  success: function(res) {
    console.debug('字体加载成功', res)
  },
  fail: function(res) {
    console.debug('字体加载失败', res)
  }
})
```

### 方案3：Canvas绘图使用系统字体

对于Canvas绘图，**强烈建议只使用系统默认字体**，避免引入外部字体。

**Canvas字体设置**：
```javascript
// 通用安全字体
ctx.font = 'bold 24px sans-serif';

// 中文优先的安全字体
ctx.font = 'bold 24px "PingFang SC", "Microsoft YaHei", sans-serif';
```

## 项目具体修复方案

### 1. 全局字体优化（app.wxss）

将全局字体设置为微信小程序安全字体栈，确保在所有设备上都能正常显示。

### 2. Canvas绘图字体优化

所有使用Canvas的页面（头像生成器、文本转图片等）：
- 移除对特殊字体的依赖
- 只使用系统默认字体
- 提供字体回退机制

### 3. 字体加粗优化

某些Android设备上font-weight不生效，可以使用-webkit-text-stroke作为备选方案。
