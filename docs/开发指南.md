## 开发流程

### 1. 新增工具

1. 在 `packages/` 目录下创建对应的工具包目录（如果不存在）
2. 在工具包目录下创建 `pages/` 目录
3. 在 `pages/` 目录下创建工具页面
4. 在 `config/tools.js` 中添加工具配置
5. 在 `app.json` 中注册工具页面

### 2. 工具配置

工具配置项说明：

```javascript
{
  id: 'tool-id',           // 工具唯一ID
  name: '工具名称',         // 工具名称
  icon: '📱',              // 工具图标
  color: 'blue',           // 工具颜色
  url: '/path/to/page',    // 工具页面路径
  categories: ['分类1', '分类2'], // 工具分类
  keywords: ['关键词1', '关键词2'], // 搜索关键词
  description: '工具描述'    // 工具描述
}
```

### 3. 代码规范

- 使用 ES6+ 语法
- 遵循 ESLint 规则
- 使用 Prettier 格式化代码
- 为关键函数添加注释

### 4. 调试

- 使用微信开发者工具进行调试
- 在控制台查看日志
- 使用 `console.log()` 打印调试信息

## 构建与发布

### 1. 构建

1. 确保所有依赖已安装：`npm install`
2. 构建项目：使用微信开发者工具的构建功能

### 2. 发布

1. 在微信开发者工具中提交代码
2. 填写版本号和更新说明
3. 提交审核
4. 审核通过后发布

## 注意事项

- 工具页面路径必须与 `app.json` 中的配置一致
- 工具ID必须唯一
- 工具分类必须在 `config/categories.js` 中定义
- 保持代码风格一致
- 确保工具功能完整且稳定
