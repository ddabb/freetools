# 小程序设计原则

## 1. 字体处理原则

### 1.1 字体选择原则
- **优先使用系统默认字体**：小程序应优先使用微信小程序默认支持的字体，确保在所有设备上的一致性
- **避免依赖特殊字体**：除非必要，避免引入外部字体，防止字体缺失和加载延迟
- **字体回退策略**：始终提供字体回退方案，确保在字体缺失时有备选方案

### 1.2 微信小程序支持的默认字体
**iOS 系统**：
- PingFang SC (苹方) - 中文
- San Francisco - 英文/数字

**Android 系统**：
- 思源黑体 - 中文
- Roboto - 英文/数字

**通用字体**：
- -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif

### 1.3 字体规范
```css
/* 推荐的字体设置 */
page {
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
}

/* 中文优先 */
.text-chinese {
  font-family: 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', sans-serif;
}

/* 英文/数字优先 */
.text-english {
  font-family: 'San Francisco', Roboto, 'Helvetica Neue', Arial, sans-serif;
}
```

## 2. Canvas 绘图字体原则

### 2.1 字体声明
- **必须先声明字体**：在 Canvas 绘图前，必须确保字体已加载完成
- **字体加载监听**：使用字体加载监听机制，确保字体可用后再渲染

### 2.2 Canvas 字体安全策略
- **使用系统字体**：Canvas 绘图优先使用系统默认字体
- **避免特殊字符**：对于需要特殊效果的文本，考虑使用图片替代
- **字号适配**：根据不同设备屏幕尺寸调整字号大小

### 2.3 Canvas 字体示例
```javascript
// Canvas 2D 字体设置
const ctx = canvas.getContext('2d');

// 设置字体（使用安全字体）
ctx.font = 'bold 24px "PingFang SC", "Microsoft YaHei", sans-serif';
ctx.fillStyle = '#333333';
ctx.textAlign = 'center';
ctx.textBaseline = 'middle';

// 绘制文本
ctx.fillText('示例文本', canvas.width / 2, canvas.height / 2);
```

## 3. 响应式设计原则

### 3.1 单位选择
- **优先使用 rpx**：微信小程序推荐使用 rpx 作为响应式单位
- **1rpx = 屏幕宽度/750**：rpx 会根据屏幕宽度自动适配
- **避免固定 px**：尽量避免使用固定像素值，除非是边框等特殊情况

### 3.2 响应式布局
```css
/* 响应式容器 */
.container {
  width: 100%;
  padding: 30rpx;
  box-sizing: border-box;
}

/* 响应式字体 */
.title {
  font-size: 36rpx;
  line-height: 1.5;
}

.content {
  font-size: 28rpx;
  line-height: 1.6;
}
```

## 4. 性能优化原则

### 4.1 资源加载
- **避免外部字体**：尽量不引入外部字体文件，减少网络请求和加载时间
- **图片优化**：使用 WebP 格式图片，控制图片大小
- **按需加载**：按需加载组件和资源

### 4.2 渲染优化
- **减少重绘**：避免频繁的 DOM 操作和样式修改
- **使用 setData 优化**：合理使用 setData，避免频繁调用
- **Canvas 优化**：合理使用 Canvas，避免过度绘制

## 5. 跨平台兼容性原则

### 5.1 系统差异处理
- **iOS 和 Android 差异**：注意两个平台的字体、渲染差异
- **屏幕尺寸适配**：使用响应式设计适配不同屏幕尺寸
- **测试验证**：在不同设备和系统版本上进行充分测试

### 5.2 兼容性检查清单
- [ ] 字体在 iOS 和 Android 上显示一致
- [ ] Canvas 绘图在不同设备上效果一致
- [ ] 响应式布局在不同屏幕尺寸上正常显示
- [ ] 性能在低端设备上流畅运行
- [ ] 所有功能在新旧系统版本上正常工作

## 6. 可维护性原则

### 6.1 代码规范
- **统一代码风格**：遵循统一的代码规范和命名约定
- **注释清晰**：关键代码添加必要的注释
- **模块化设计**：合理拆分代码模块，提高复用性

### 6.2 文档维护
- **设计文档**：维护完整的设计文档和规范
- **变更记录**：记录重要的设计变更和优化
- **测试文档**：维护测试用例和测试结果

## 7. 用户体验原则

### 7.1 视觉一致性
- **统一设计风格**：保持整个应用的视觉风格一致
- **色彩规范**：使用统一的色彩方案
- **间距规范**：保持统一的间距比例

### 7.2 交互体验
- **反馈及时**：用户操作要有及时的反馈
- **加载状态**：显示清晰的加载状态
- **错误处理**：友好的错误提示和处理方式

## 8. 安全性原则

### 8.1 内容安全
- **内容审核**：确保用户生成的内容符合微信平台规范
- **敏感词过滤**：对用户输入进行敏感词过滤
- **数据安全**：保护用户数据安全，合理使用用户信息

### 8.2 合规性
- **遵守平台规范**：严格遵守微信小程序平台规范
- **隐私保护**：保护用户隐私，遵循相关法律法规
