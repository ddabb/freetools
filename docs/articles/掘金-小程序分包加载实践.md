# 69个工具怎么塞进一个小程序？我踩了这些坑

---

做小程序最头疼的事：工具太多，包体积不够。

主包 2MB，总包 20MB。69 个工具全塞进去，早就超了。

这篇文章记录我是怎么用**分包策略**解决问题的，以及踩过的坑。

---

## 小程序包体积就这回事

| 类型 | 限制 |
|------|------|
| 主包 | 2MB |
| 单个分包 | 2MB |
| 总包 | 20MB |

每个工具页面有 `.wxml`、`.wxss`、`.js`、`.json` 四个文件，加上公共组件、图片资源，69 个工具全放主包肯定超。

---

## 怎么分

按功能类型划分：

- **math**：数学工具
- **text**：文本工具
- **life**：生活工具
- **unit**：单位换算
- **financial**：金融工具
- **converter**：格式转换
- 还有 health、travel、developer、design、food、knowledge

主包只保留首页、知识库等核心页面，800KB 左右。

```json
{
  "pages": ["pages/index/index", "pages/about/about"],
  "subPackages": [
    { "root": "packages/math", "pages": [...] },
    { "root": "packages/text", "pages": [...] },
    ...
  ]
}
```

## 预加载，让体验接近秒开

光分包不够，用户点进去还要等加载。

用**预加载**解决：打开首页时，后台先把高频工具的分包拉下来。

```json
{
  "preloadRule": {
    "pages/index/index": {
      "network": "all",
      "packages": ["packages/life", "packages/math"]
    }
  }
}
```

`network: "all"` 表示 WiFi 和移动网络都预加载。如果想省流量可以改成 `"wifi"`，但体验会打折扣。

## 还有个坑：数据太大

成语接龙有几万条成语，总不能打包进小程序吧？

**数据放 CDN，按需拉取，本地缓存。**

```javascript
async function loadIdioms() {
  const cached = wx.getStorageSync('idioms');
  if (cached) return cached;
  
  const res = await wx.request({
    url: 'https://你的CDN/idioms.json'
  });
  
  wx.setStorageSync('idioms', res.data);
  return res.data;
}
```

第一次用拉一下，后面直接读缓存。

## 踩过的坑

### 分包不能互相引用

分包之间是隔离的，A 分包不能引用 B 分包的组件。

```javascript
// ❌ 报错
{ "usingComponents": { "my-comp": "/packages/text/components/..." } }

// ✅ 正确：引用主包的组件
{ "usingComponents": { "my-comp": "/components/my-comp" } }
```

公共资源统一放主包。

### 独立分包限制多

如果想从分享链接直接打开某个工具，需要设为独立分包：

```json
{
  "root": "packages/share-tool",
  "independent": true
}
```

但独立分包不能引用主包资源，按需使用。

## 总结

核心思路就四点：

1. **主包只放必要的** — 首页、公共组件
2. **按功能分包** — 哪个分包用到再加载
3. **数据放 CDN** — 大文件不打包
4. **预加载高频分包** — 常用工具秒开

「随身工具宝」69 个工具能用这个方案，你的小程序大概率也能用。
