## Dancing Links 是什么

Dancing Links（简称 DLX）是 Knuth 提出的一种**优化后的回溯算法**，专门用来解精确覆盖问题。

普通回溯是遍历所有可能性，DLX 用**双向链表**把数据结构组织起来，剪枝效率更高——选了一个数字，一行相关的候选瞬间清空，不用逐个检查。

具体原理不展开了，总结就是：**更快，毫秒级无压力**。

---

## 核心思路

把数独问题转化为**精确覆盖问题**：
- 81 个格子 × 9 个数字 = 729 种可能
- 加上行、列、宫格的约束条件

Dancing Links 用双向链表组织这些约束，选中一个节点后，相关行、列、宫格的候选自动移除，效率比普通回溯高得多。

```javascript
// Dancing Links 核心结构
class DLX {
  constructor() {
    this.header = { right: null, down: null };
    // ... 初始化 4 类约束：格子、行、列、宫
  }
  
  // 搜索
  search(k) {
    if (this.header.right === this.header) {
      // 找到解
      return true;
    }
    
    // 选择约束最少的一列
    const c = this.selectColumn();
    
    // 遍历该列的所有行
    for (let r = c.down; r !== c; r = r.down) {
      this.cover(r.column);
      
      if (this.search(k + 1)) return true;
      
      this.uncover(r.column); // 回溯
    }
    return false;
  }
}
```

### 约束系统

数独需要 4 类约束：
1. **格子约束**：每个格子必须填一个数字
2. **行约束**：每行 1-9 各出现一次
3. **列约束**：每列 1-9 各出现一次
4. **宫约束**：每宫 1-9 各出现一次

总共 324 个约束，用矩阵表示就是 81×9 行 × 324 列。

## 还能生成题目

反过来想就行：**先解一个完整的数独，再随机挖掉一些格子**。

```javascript
function generate(holes = 45) {
  // 1. 生成完整解
  const board = createEmptyBoard();
  solve(board);
  
  // 2. 保存答案
  const solution = board.map(row => [...row]);
  
  // 3. 随机挖空
  const positions = shuffle([...81个位置]);
  for (let i = 0; i < holes; i++) {
    const [r, c] = positions[i];
    board[r][c] = 0;
  }
  
  return { puzzle: board, solution };
}
```

挖掉 35-40 个格子是简单难度，40-50 个是中等，50-55 个是困难。

## 还能做什么

Dancing Links 不仅能解数独。

- **N 皇后问题**
- **拉丁方阵**
- **数独生成器**（先生成完整解，再挖空）
- 各种精确覆盖问题

核心思路都是一样的：把问题建模成矩阵，用双向链表高效剪枝。
