# 成语接龙开发记录

> 记录成语接龙功能从零到上线的完整过程，包含需求、技术方案、踩坑实录。

## 一、需求起源

2026年清明假期，老婆突然想玩成语接龙，但没有好的工具辅助。于是决定用 AI 做一个小程序来解决这个问题。

核心需求：
- **查询模式**：输入任意成语，查看所有可接龙的下联
- **对战模式**：人机对战成语接龙，AI 自动判断输赢
- **接龙链模式**：输入起始成语，自动生成一条最长接龙链

## 二、文件结构

成语接龙功能已拆分为三个独立页面，共用数据服务模块：

```
packages/life/pages/
├── idiom-query/      ← 查询模式（查询成语接龙下联）
├── idiom-chain/       ← 接龙链模式（DFS 生成接龙链）
├── idiom-battle/      ← 对战模式（人机成语接龙）
└── utils/
    └── idiom-data-service.js  ← 公用数据服务（CDN 加载 + 缓存管理）
```

**数据服务模块**（`idiom-data-service.js`）统一处理：
- CDN 数据加载（`idiom_first_full_index.json`、`idiom_last_index.json`）
- 本地缓存（Storage）
- 成语查询（`querySolitaire`）
- 接龙链生成（`generateChain`）
- 缓存清理（`clearCache`、`clearLogs`）

### 2.1 数据源

成语数据来自开源项目 `by-syk/chinese-idiom-db`，包含约 12,970 条成语。使用 `pinyin-pro` 处理拼音，构建首字/尾字索引，按需加载 letter 文件获取释义。

## 三、技术方案

### 3.1 CDN 分发架构

```
idiom-first-full-index.json  (~200KB)  →  首字完整拼音索引
idiom-last-index.json        (~200KB)  →  尾字完整拼音索引
letter/a.json ~ z.json       (~按需)    →  字母分组，含释义和出处
```

所有数据走 CDN 静态分发，无需服务端。

### 3.2 缓存策略

- 索引数据缓存 7 天（变化极小）
- Storage Key：`cdn_idiom_index_data`、`cdn_idiom_index_ts`
- 另有 `cdn_idiom_data_logs` 记录操作日志（最多 100 条）
- 所有 CDN 相关 key 均以 `cdn_` 开头，`app.js` 版本升级时会自动清理
- 提供 `clearCache()` 和 `clearLogs()` 方法供外部调用

### 3.3 对战模式算法

```
对战流程：
1. 随机选一个四字成语作为龙头
2. 玩家需要接出以龙尾字开头的成语
3. 支持汉字接龙（严格匹配）或同音接龙（拼音匹配）
4. AI 随机从候选池选一个成语应战
5. 60 秒倒计时，超时判负
```

### 3.4 接龙链 DFS 算法

使用深度优先搜索生成接龙链，关键参数：
- 最大长度：默认 10，最多 50
- 每次 DFS 最多尝试 15 个候选（避免爆炸）
- 使用 `setTimeout(0)` 实现流式输出，边搜边显示

## 四、踩坑实录

### 4.1 流式输出的坑

最初的 DFS 实现是等全部搜完才显示结果，用户等待时间过长。

**解决方案**：每次到达终止条件立即 `setData` 更新，并使用 `setTimeout(0)` 让出主线程，让 UI 有机会渲染。

```javascript
setTimeout(() => {
  dfs(nextPy, depth + 1, path, used);
}, 0);
```

新增 `_chainCancelled` 标志位，支持取消进行中的搜索。

### 4.2 AI 生成 UI 的"word感"

用 AI 生成的小程序界面有很强的 Word 文档感——方方正正、毫无层次、按钮颜色浅得像没墨。

**解决方案**：逐步优化 CSS，增加圆角、渐变、阴影，按钮文字改深色，提升对比度。

### 4.3 WXSS 编译异常

开发过程中遇到编译器热重载缓存产生的异常字符，导致 tab-content 区域被推出了屏幕外，查询按钮完全点不到。

**解决方案**：将内联样式迁移到 WXSS 文件，统一管理，避免编译器干扰。
