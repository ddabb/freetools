# 常见问题排查手册

> 记录 FreeTools 开发过程中遇到的各种问题及其解决方案，持续更新。

---

## 一、小程序无法加载

### 1.1 页面 404

**表现**：进入页面时报 "页面不存在"

**排查步骤**：
1. 检查 `app.json` 的 `subPackages` 是否注册了该页面路径
2. 检查页面文件是否存在于对应目录
3. 检查 `config/tools.js` 中的 `url` 路径是否正确

**解决**：
检查 `app.json` 的 `subPackages` 是否注册了该页面路径，确保路径存在（如成语接龙有三个独立页面：`idiom-query`、`idiom-chain`、`idiom-battle`）。

### 1.2 分包加载失败

**表现**：进入分包页面时提示加载失败

**排查**：
1. 确认分包路径正确
2. 检查分包目录下文件是否完整（4个文件都存在）
3. 重启微信开发者工具，清除缓存

## 二、数据加载问题

### 2.1 CDN 数据不更新

**表现**：更新了 CDN 文件，但小程序显示的还是旧数据

**原因**：CDN 缓存未刷新 / 本地 Storage 缓存未过期

**解决方案**：成语接龙的缓存清理推荐调用 `idiom-data-service` 的 `clearCache()` 方法，或手动清除：
```javascript
wx.removeStorageSync('cdn_idiom_index_data');
wx.removeStorageSync('cdn_idiom_index_ts');
wx.removeStorageSync('cdn_idiom_data_logs');
```

**根本解决**：等待 CDN 刷新（通常 1-5 分钟）或切换分支版本号

### 2.2 CDN 请求报错

**常见错误**：
- `domain not found`：域名拼写错误
- `ssl handshake failed`：HTTPS 证书问题
- `timeout`：网络超时或请求超时

**排查**：
1. 在浏览器直接访问 CDN 链接测试
2. 检查 jsDelivr 是否正常：`https://www.jsdelivr.com`
3. 检查 GitHub 仓库是否存在

### 2.3 数据为空

**排查**：
1. 检查 CDN JSON 文件是否有效（浏览器直接打开）
2. 检查 `JSON.parse()` 是否报错
3. 检查数据 key 是否匹配

## 三、样式问题

### 3.1 组件样式不生效

**排查步骤**：
1. 检查 `.wxss` 文件是否正确命名
2. 检查选择器是否正确
3. 检查优先级（内联样式 > class）
4. 检查是否被覆盖

**常见错误**：
```wxss
/* ✗ 选择器拼写错误 */
.crad { }     // 应该是 .card

/* ✗ 用了中文字符 */
page｛ ｝      // 应该是 page { }

/* ✗ 路径引用错误 */
@import './card.wxss';  // 路径不存在
```

### 3.2 WXSS 编译报错

**表现**：开发者工具提示 WXSS 编译错误

**常见原因**：
- 意外的 `\` 反斜杠（编译器热重载 bug）
- 中文字符出现在 CSS 语法中
- 缺少闭合括号

**解决**：
1. 重启微信开发者工具
2. 删除报错文件，重新创建
3. 检查是否有非法字符

### 3.3 内联样式与 WXSS 冲突

**排查**：检查元素是否有 `style` 属性

**解决**：将内联样式迁移到 WXSS

## 四、逻辑问题

### 4.1 setData 不生效

**常见原因**：

| 原因 | 表现 | 解决 |
|------|------|------|
| 直接修改 `this.data` | 数据变了但视图不更新 | 必须用 `setData` |
| 对象引用问题 | 修改后所有引用都变 | 使用展开运算符 `...` |
| 异步时序问题 | 数据还没更新就用 | 使用回调或 async/await |

```javascript
// ✗ 错误：直接修改
this.data.result = newValue;

// ✓ 正确：使用 setData
this.setData({ result: newValue });

// ✗ 错误：对象引用问题
const list = this.data.list;
list.push(newItem);  // list 和 this.data.list 指向同一对象
this.setData({ list });

// ✓ 正确：创建新引用
const list = [...this.data.list, newItem];
this.setData({ list });
```

### 4.2 事件不触发

**排查**：
1. 检查是否用 `bindtap` 而非 `onTap`（小程序不支持 on 开头的事件）
2. 检查方法名是否在 `Page({})` 中定义
3. 检查 `data-*` 属性是否正确绑定

```html
<!-- ✗ 错误：方法未定义 -->
<view bindtap="onClik">   <!-- 拼写错误 -->

<!-- ✓ 正确 -->
<view bindtap="onClick">
```

### 4.3 死循环/栈溢出

**表现**：小程序卡死或提示栈溢出

**常见原因**：
- `setData` 在 `setData` 的回调里无限调用
- DFS 递归层数过深
- `Page` 外部有执行代码

```javascript
// ✗ 危险：递归无限调用
_doChain() {
  this._doChain();  // 无限递归
}

// ✓ 安全：设置终止条件
_dfs(depth) {
  if (depth > MAX_DEPTH) return;
  // ...
  this._dfs(depth + 1);
}
```

## 五、性能问题

### 5.1 页面卡顿

**排查**：
1. 检查 `setData` 是否传了大量数据
2. 检查是否有复杂计算阻塞主线程
3. 检查长列表是否使用了虚拟列表

**优化建议**：
```javascript
// ✗ 低效：每次 setData 传整个列表
this.setData({ results: hugeArray });

// ✓ 高效：只更新变化的部分
this.setData({ [`results[${index}]`]: newItem });
```

### 5.2 图片加载慢

**解决**：
1. 压缩图片大小（单张 < 200KB）
2. 使用 CDN 加速
3. 启用图片懒加载

### 5.3 Canvas 绘图闪烁

**原因**：频繁重绘导致闪烁

**解决**：使用离屏 Canvas + `drawImage` 一次性绘制

## 六、Canvas 图片生成问题

### 6.1 字体不显示

**原因**：字体未加载完成就开始绘制

**解决**：
```javascript
// 先加载字体
wx.loadFont({
  family: 'MyFont',
  source: '...',
  success: () => {
    // 字体加载成功后再绘制
    this._draw();
  }
});
```

### 6.2 导出图片模糊

**原因**：未处理高清屏（dpr）

**解决**：
```javascript
// 创建 Canvas 时指定尺寸
<canvas type="2d" style="width: {{width * dpr}}px; height: {{height * dpr}}px; transform: scale({{1/dpr}});">

// dpr = wx.getSystemInfoSync().pixelRatio
```

### 6.3 导出失败

**排查**：
1. Canvas 是否在视口内（不可见 Canvas 无法导出）
2. 检查 `canvasToTempFilePath` 参数
3. 检查是否有未捕获的绘制错误

## 七、调试技巧

### 7.1 常用调试代码

```javascript
// 输出到控制台
console.log('[idiom] data:', this.data);

// 显示调试信息
wx.showModal({
  title: '调试',
  content: JSON.stringify(this.data),
});

// 临时存储查看
wx.setStorageSync('debug', this.data);
```

### 7.2 清除所有缓存

```javascript
// 清除所有本地缓存（测试用）
wx.clearStorageSync();
```

### 7.3 快速复现问题

1. 清除小程序缓存：`设置 → 缓存 → 清除`
2. 重启微信开发者工具
3. 删除 `data/` 和 `__fileCaches__` 目录

## 八、报错代码对照

| 错误信息 | 含义 | 常见原因 |
|----------|------|----------|
| `module is not defined` | 模块未定义 | JS 语法错误，Page 外有执行代码 |
| `page is not defined` | 页面未定义 | app.json 未注册 |
| `component is not found` | 组件未找到 | usingComponents 配置错误 |
| `expect END tag` | XML 解析错误 | WXML 标签未闭合 |
| `Unexpected token` | 语法错误 | JSON 格式错误或 JS 语法错误 |
