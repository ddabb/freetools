# FreeTools 项目文档目录

本目录存放 FreeTools 微信小程序项目的所有文档。

## 文档列表

### 1. 项目提案.html
- **内容**：项目提案文档
- **说明**：包含项目概述、核心功能、页面设计、用户流程等详细信息
- **用途**：用于项目立项和需求评审

### 2. OPTIMIZATION_SUMMARY.md
- **内容**：优化总结文档
- **说明**：记录已完成的高优先级和中优先级优化任务
- **用途**：了解项目的优化历程和优化效果

### 3. README.md
- **内容**：文档说明
- **说明**：本文档，说明文档目录的用途和注意事项

## 文档管理原则

### 文档位置
- 所有项目文档统一存放在 `docs/` 目录
- 文档不会被打包到小程序中
- 便于团队协作和文档维护

### 文档格式建议
- 技术文档：使用 `.md` 格式
- 设计文档：可以使用 `.html` 或 `.md` 格式
- 图片资源：建议放在 `docs/images/` 子目录

### 文档命名规范
- 使用英文或中文命名，保持清晰易懂
- 避免使用特殊字符和空格
- 示例：`项目提案.html`、`OPTIMIZATION_SUMMARY.md`、`API文档.md`

## 打包规则说明

根据 `project.config.json` 配置，以下规则确保文档不会被打包：

```json
{
  "packOptions": {
    "ignore": [
      "docs/**/*",
      ".codebuddy/**/*",
      "项目提案.html",
      "*.md"
    ]
  }
}
```

**说明：**
- `docs/**/*` - 排除 docs 目录下所有文件
- `.codebuddy/**/*` - 排除开发工具目录
- `项目提案.html` - 排除根目录的 HTML 文件
- `*.md` - 排除所有 Markdown 文件

## 注意事项

1. 文档更新后，建议在 `docs/` 目录中保持版本控制
2. 重要变更应在 `OPTIMIZATION_SUMMARY.md` 中记录
3. 技术文档建议包含示例代码和使用说明
4. 设计文档应包含 UI 图和交互说明
