## 文档管理原则

### 文档位置
- 所有项目文档统一存放在 `docs/` 目录
- 文档不会被打包到小程序中
- 便于团队协作和文档维护

### 文档格式建议
- PRD/提案：使用 `.md` 格式
- 设计文档：使用 `.md` 格式
- 图片资源：建议放在 `docs/images/` 子目录

### 命名规范
- 有效文档：`公众号配图增强PRD.md`
- 历史文档：`tab-design-proposal_历史.md`（加 `_历史` 后缀）
- 避免特殊字符，使用中文或英文均可

## 打包规则说明

根据 `project.config.json` 配置，以下规则确保文档不会被打包：

```json
{
  "packOptions": {
    "ignore": [
      "docs/**/*",
      ".codebuddy/**/*"
    ]
  }
}
```

## 注意事项

1. 文档更新后，建议在 changelog 中记录
2. 重要变更应在 `优化总结.md` 中追加
3. 新增 PRD 文档放在 `docs/` 根目录
4. 过期的历史文档加 `_历史` 后缀保留，不删除
