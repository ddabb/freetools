## 文档管理原则

- 所有项目文档统一存放在 `docs/` 目录
- 文档不会被打包到小程序中
- 新增 PRD/提案 使用 `docs/` 根目录，格式 `.md`
- 过期的历史文档加 `_历史` 后缀，不删除

## 打包规则

`project.config.json` 中已配置排除 docs 目录：

```json
"packOptions": {
  "ignore": ["docs/**/*", ".codebuddy/**/*"]
}
```

## 注意事项

1. 文档更新后，在 changelog 中记录
2. 技术文档建议包含示例代码
3. PRD 文档使用 Markdown，表格+列表为主
