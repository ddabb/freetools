## 二、方案一：工具箱定位（轻量版）

适合：工具导向，用户来就是找工具用

| Tab | 名称 | 功能 |
|-----|------|------|
| 1 | 首页 | 工具分类导航 |
| 2 | 收藏 | 我的收藏工具 |
| 3 | 关于 | 应用信息 |

**优点**：极简，用户路径清晰

### 免费 API 清单

| 功能 | API | 免费限制 | 备注 |
|------|-----|----------|------|
| 🌤️ 天气 | wttr.in | 无限制 | 无需 key |
| 💬 一言 | hitokoto.cn | 无限制 | 无需 key |
| 🎯 今日一言 | 随机古诗词 API | 无限制 | 无需 key |
| 📅 今日信息 | 本地计算 | 0 | 不需要 API |
| 🤖 AI 对话 | 需付费 | - | 暂不接入 |

### 数据存储设计（纯本地）

```javascript
// Storage Key 设计

  key: 'userData',

    ],
    toolUsageCount: {                          // 工具使用次数
      'text-to-png': 15,
      'qrcode': 8

    firstVisit: 1710000000                     // 首次访问时间
  }

```

## 四、方案三：文章 Tab 详细设计（号主版）

适合：公众号运营者，想通过工具引流内容

### 核心定位

```
目的：通过工具引流 → 用户阅读文章 → 关注公众号
```

### 内容来源（无后端方案）

| 方案 | 来源 | 更新方式 | 难度 |
|------|------|----------|------|
| A | 静态 JSON 文件 | 手动更新 | ⭐ 简单 |
| B | 公众号图文消息 API | 自动同步 | ⭐⭐ 需配置 |
| C | RSS 订阅 | 自动抓取 | ⭐⭐⭐ 需 RSS 服务 |
| D | 混合方案 | 静态 + 动态 | ⭐⭐ 中等 |

## 七、最终建议

### 推荐方案组合

#### 短期（快速上线）

```
首页 / 我的 / 关于
```

其中「我的」包含：
- 🌤️ 今日天气（免费 API）
- 💬 一言（免费 API）
- ⭐ 我的收藏
- 🕐 最近使用
- 📈 使用统计

#### 中期（增加内容）

```
首页 / 文章 / 关于
```

其中「文章」包含：
- 号主文章列表
- 工具使用教程
- 公众号引流入口

#### 长期（完整生态）

```
首页 / 文章 / 我的 / 关于
```

---

### 决策要点

在选择方案时，考虑以下问题：

1. **你的核心目标是什么？**
   - 纯工具 → 方案一
   - 用户运营 → 方案二
   - 公众号引流 → 方案三
   - 体验至上 → 方案四

2. **你愿意投入多少时间？**
   - 快速上线 → 方案一/二 + 静态 JSON
   - 持续运营 → 方案三 + 公众号 API

3. **是否有公众号运营需求？**
   - 有 → 方案三
   - 无 → 方案二
