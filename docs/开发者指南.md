## 2. 页面文件规范

每个页面由 4 个文件组成，放在对应包目录下：

```
packages/<包名>/pages/<页面名>/<页面名>.(js|json|wxml|wxss)
```

**示例：** 新增一个"随机数"页面

```
packages/math/pages/random-number/random-number.js
packages/math/pages/random-number/random-number.json
packages/math/pages/random-number/random-number.wxml
packages/math/pages/random-number/random-number.wxss
```

**文件命名规则：**
- 目录名 = 文件名（不含扩展名）
- 全部小写，单词用连字符 `-` 分隔
- 避免中文目录名

## 3. 分包注册（app.json）

每个新工具包需要在 `app.json` 的 `subPackages` 中注册：

```json
{
  "root": "packages/bobing",
  "name": "bobing",
  "pages": [
    "pages/bobing/bobing/bobing"
  ]
}
```

**注意：** 微信限制主包 + 分包总大小不超过 20MB，分包建议按需加载。

## 4. 工具入口注册（config/tools.js）

新工具需要在 `tools.js` 的 `tools` 数组中添加配置块：

```js
{
  id: 'random-number',
  name: '随机数生成',
  icon: '🎲',
  color: 'blue',
  url: '/packages/math/pages/random-number/random-number',
  categories: ['数学工具'],
  keywords: ['随机', '随机数', '抽奖', '概率'],
  description: '生成指定范围内的随机整数'
},
```

详见 `工具添加规范.md`。

## 5. CDN 数据加载规范

项目使用 jsDelivr CDN + GitHub 存储静态数据，实现免发布更新。

### 5.1 数据文件存放路径

```
docs/data/<功能名>/<数据文件>.json
```

### 5.2 CDN 路径格式

```
https://cdn.jsdelivr.net/gh/ddabb/FreeToolsPuzzle@main/docs/data/<功能名>/<文件>.json
```

### 5.3 加载代码模板

```js
const CDN_BASE = 'https://cdn.jsdelivr.net/gh/ddabb/FreeToolsPuzzle@main';
const DATA_URL = `${CDN_BASE}/docs/data/changelog.json`;

// 带缓存的加载函数
async function loadData(url, cacheKey, maxAge = 24 * 60 * 60 * 1000) {
  const cached = wx.getStorageSync(cacheKey);
  const now = Date.now();

  if (cached && (now - cached.timestamp) < maxAge) {
    return cached.data;
  }

  try {
    const res = await wx.request({ url });
    if (res.data) {
      wx.setStorageSync(cacheKey, {
        data: res.data,
        timestamp: now
      });
      return res.data;
    }
  } catch (e) {
    // 网络失败，尝试使用缓存
    if (cached) return cached.data;
  }

  return null;
}
```

### 5.4 已有 CDN 数据

| 数据 | 路径 | 用途 |
|------|------|------|
| 版本日志 | `docs/data/changelog.json` | changelog 页面 |
| 数独题目 | `docs/data/sudoku-presets.json` | 数独求解器 |
| 24点题目 | `docs/data/24point-questions.json` | 24点工具 |
| 文案数据 | `docs/data/wordbank/` | 文案工具 |

## 6. 图片生成规范（utils/imageGenerator.js）

文案生图、公众号配图等使用 Canvas 绘制后导出。

### 6.1 导出格式

- 文件类型：`png`
- 质量：`quality: 1`（生成时）/ `quality: 0.8`（预览时）
- 分辨率：按设计要求，建议宽度 ≤ 750px

### 6.2 水印格式（已修改）

```js
// 来源水印格式（使用 @ 而非 ——）
const text = '@ ' + from;
```

### 6.3 字体配置

```js
ctx.font = `bold ${size}px "Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif`;
```


## 8. Git 协作规范

- **commit 前先 pull**：避免覆盖他人代码
- **commit 规范**：`feat: 新增XX功能` / `fix: 修复XX问题` / `docs: 更新文档`
- **push 前确认**：外部操作（push/deploy）必须先问彪哥，不自作主张
- **敏感信息**：私密内容不上传 git

## 9. 微信审核注意事项

| 风险点 | 规避方式 |
|--------|---------|
| 工具名称含"游戏" | 用"博饼"等文化词，不用"游戏" |
| 涉及随机/抽奖 | 纯娱乐，不涉及积分/排名/奖品 |
| 功能描述夸大 | 客观描述功能，不夸大宣传 |
| 分类名含"游戏" | 归入"生活工具"，不放"游戏"分类 |

详见 `小程序设计原则.md`。
