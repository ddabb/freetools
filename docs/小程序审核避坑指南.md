# 小程序审核避坑指南

> 记录 FreeTools 在微信小程序审核中遇到的各类问题和解决方案，持续更新。

## 一、审核机制概述

微信小程序审核主要关注：
1. **功能合规**：是否符合小程序类目规范
2. **内容合规**：是否存在违规内容
3. **体验规范**：是否影响用户体验

审核周期：通常 1-7 个工作日

## 二、高频踩坑点

### 2.1 命名类

| 踩坑 | 规避方案 |
|------|----------|
| 工具名含"游戏" | 用"博饼"等文化词替代 |
| 工具名含"彩票" | 用"号码生成"等中性词 |
| 分类名含"游戏" | 归入"生活工具"等非敏感分类 |
| 工具名含"赌博"相关 | 避免任何博彩暗示 |

**案例**：早期将"今天吃什么"分类为"娱乐游戏"，审核被拒。后改为"生活工具"分类通过。

### 2.2 功能类

| 功能描述 | 审核风险 | 规避方案 |
|----------|----------|----------|
| 涉及随机/抽奖 | 可能被认定博彩 | 明确为"娱乐工具"，不涉及奖品 |
| 排行榜 | 需社交类目 | 去掉排行榜或申请对应类目 |
| 用户生成内容 | 需内容审核机制 | 先做UGC白名单机制 |
| 支付相关 | 需申请支付类目 | 使用虚拟支付需谨慎 |

### 2.3 内容类

| 内容类型 | 风险等级 | 说明 |
|----------|----------|------|
| 版权内容 | 高 | 避免使用未授权成语释义、诗词等 |
| 政治敏感 | 极高 | 绝对禁止 |
| 医疗健康 | 高 | 需提供资质或加免责声明 |
| 金融理财 | 高 | 需对应资质，明确非投资建议 |

## 三、审核话术优化

### 3.1 功能描述模板

❌ 不推荐：
> "随机抽奖工具，公平公正"

✅ 推荐：
> "随机选择工具，帮助您在多个选项中公平决策，适用于聚餐选餐厅、活动分组等场景。纯娱乐用途，不涉及任何奖品或金钱。"

### 3.2 类目选择建议

| 实际功能 | 建议类目 |
|----------|----------|
| 工具集合 | 工具-效率 |
| 百科知识 | 教育-在线教育 |
| 生活服务 | 生活服务-综合服务 |
| 文本处理 | 工具-办公 |

## 四、申诉流程

如果审核被拒，可以通过以下方式申诉：

1. **阅读拒绝原因**：审核详情通常会说明具体问题
2. **针对性修改**：根据原因修改描述或功能
3. **申诉入口**：微信公众平台 → 版本管理 → 审核详情 → 申诉
4. **准备材料**：截图、说明文档、功能演示视频

### 申诉话术模板

```
尊敬的审核团队：

感谢审核。针对您提出的 [具体问题]，我方已做如下调整：

1. [调整项1]
2. [调整项2]

[补充说明]

盼复，谢谢。
```

## 五、上线后合规

审核通过不代表可以随意修改，以下操作需要重新审核：
- 基础功能变更
- 新增功能模块
- 修改小程序类目
- 重大UI调整

日常运营维护（Bug修复、小优化）不需要重新审核。

## 六、常见审核结果

| 审核结果 | 含义 | 处理方式 |
|----------|------|----------|
| 通过 | 功能已上线 | 正常发布 |
| 待审核 | 排队中 | 等待 |
| 灰度中 | 分阶段放量 | 观察数据 |
| 被拒 | 功能不符合规范 | 根据原因修改后申诉 |

## 七、自检清单

发布前逐项确认：

- [ ] 功能描述与实际一致
- [ ] 无任何"游戏""赌博""彩票"等敏感词
- [ ] 截图真实，无P图
- [ ] 用户协议/隐私政策已配置（如需）
- [ ] 无未授权的第三方内容
- [ ] 类目与核心功能匹配
