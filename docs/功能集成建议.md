# FreeTools 功能集成建议

> 创建时间：2026-04-17
> 目标：提升用户粘性、增强产品价值、扩展使用场景
> 状态：待评审

---

## 一、AI 能力集成 🤖

### 1.1 AI 文案助手增强

**现状：** 文案生成页面已接入 CDN 数据，但内容固定、无个性化

**建议：**
1. **AI 续写/改写** - 接入 AI API（如 DeepSeek、通义千问），支持：
   - 文案续写：用户输入开头，AI 续写剩余内容
   - 文案改写：一键改写成不同风格（正式/活泼/文艺）
   - 文案润色：优化语法和表达

2. **智能纠错** - 对用户输入的内容进行：
   - 错别字检测和纠正建议
   - 标点符号规范检查
   - 敏感词过滤提示

**技术方案：**
```js
// utils/ai-service.js
const AI_API = 'https://api.deepseek.com/v1/chat/completions';

exports.rewriteText = async (text, style) => {
  const prompt = `请将以下文案改写为${style}风格，保持原意：\n${text}`;
  // 调用 API...
};
```

**成本预估：** DeepSeek API 约 0.001 元/千 token，日均 1000 次调用约 10 元

**优先级：** 🟡 中（需评估 API 成本）

---

### 1.2 AI 知识问答

**现状：** 知识库基于静态文章，无法回答个性化问题

**建议：**
- 在知识详情页添加"AI 解读"按钮
- 基于当前文章内容，用户可提问相关问题
- AI 结合文章上下文给出解答

**示例场景：**
- 用户阅读"医保报销指南" → 点击"AI 解读"→ 问"异地就医怎么报销？"
- AI 基于文章内容 + 通用知识回答

**技术方案：**
```js
// RAG 架构：文章内容作为上下文
const context = article.content;
const prompt = `基于以下内容回答问题：\n${context}\n\n问题：${question}`;
```

**优先级：** 🟢 低（需完善知识库内容后再考虑）

## 二、语音交互能力 🎤

### 2.1 语音输入

**现状：** 所有工具仅支持键盘输入

**建议：**
为以下场景添加语音输入：
- 文案生图：语音描述想要的图片
- 文案生成：语音口述需求
- 成语查询：语音说出成语
- 计算器：语音读出算式

**技术方案：**
```js
// 使用微信原生语音识别
wx.startRecord({
  success(res) {
    const tempFilePath = res.tempFilePath;
    // 调用语音识别 API
    wx.translateVoice({
      lfrom: 'zh_CN',
      tempFilePath,
      success(res) {
        console.log(res.translatedText); // 识别结果
      }
    });
  }
});
```

**成本：** 微信语音识别免费，每月 10000 次调用

**优先级：** 🟡 中（提升用户体验）

---

### 2.2 语音播报

**现状：** 知识内容仅可阅读，无音频版本

**建议：**
- 知识详情页添加"朗读"按钮
- 使用 TTS（文字转语音）播报文章内容
- 支持调节语速、暂停/继续

**技术方案：**
```js
// 方案 A：微信插件 TTS
const plugin = requirePlugin('WechatSI');
plugin.textToSpeech({
  content: article.content,
  success(res) {
    const audio = wx.createInnerAudioContext();
    audio.src = res.filename;
    audio.play();
  }
});

// 方案 B：CDN 预生成音频（成本更低）
// 构建时使用 Edge TTS 生成音频，上传 CDN
```

**成本：** 微信同声传译插件免费；预生成音频零运行成本

**优先级：** 🟢 低（场景有限）

### 3.2 使用历史增强

**现状：** 首页有"最近使用"但仅显示 10 条

**建议：**
1. **独立历史页面** - 点击"最近使用"进入完整历史
2. **分类历史** - 按工具类型分组展示
3. **历史搜索** - 搜索历史记录
4. **历史统计** - 使用频率排行、使用时间分布图

**数据结构：**
```js
// Storage
{
  usageHistory: [
    { toolId: 'sudoku', time: 1713321600000, params: {...} },
    // ...
  ]

```

**优先级：** 🟡 中

---

### 3.3 个性化推荐

**现状：** 首页工具按固定顺序 + 频率排序

**建议：**
基于用户行为推荐工具：
- **协同过滤**：用户 A 用了工具 X、Y，用户 B 也用了 X → 推荐 Y 给 B
- **时间推荐**：工作日早上推荐"倒计时""汇率换算"，晚上推荐"文案生成"
- **场景推荐**：检测到用户在"计算"类工具停留久 → 推荐更多计算工具

**技术方案：**
```js
// 简单版：基于时间推荐
const hour = new Date().getHours();
if (hour >= 9 && hour <= 12) {
  // 推荐工作类工具
  recommendTools = ['countdown', 'exchange-rate', 'calculator'];
}
```

**优先级：** 🟢 低（需积累用户数据）

### 5.2 数据导出

**现状：** 工具结果无法保存

**建议：**
为以下场景添加导出功能：
- **文案生成**：导出为 Word/PDF
- **数独题目**：导出为图片打印
- **汇率记录**：导出为 Excel
- **知识文章**：导出为 PDF 收藏

**技术方案：**
```js
// 使用 wx.saveFile 保存到本地
// 或使用云开发生成文件

// Excel 导出
const xlsx = require('node-xlsx');
const buffer = xlsx.build([{ name: '数据', data: rows }]);
wx.saveFile({
  tempFilePath: buffer,
  success(res) {
    wx.showToast({ title: '已保存' });
  }
});
```

**优先级：** 🟡 中

## 十二、风险与注意事项

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| AI API 成本超预算 | 功能下线 | 设置每日调用上限、用户配额 |
| 云开发费用超支 | 功能受限 | 监控用量、设置预算告警 |
| 功能过多导致主包过大 | 审核不通过 | 新功能放入分包、懒加载 |
| 用户数据隐私 | 合规风险 | 明确告知用户、提供删除选项 |
| 第三方 API 不稳定 | 功能不可用 | 多服务商备份、优雅降级 |
