# 数墙 (Nurikabe) 游戏规则详解

## 一、游戏背景

数墙是一种逻辑解谜游戏，由日本Nikoli公司发明。与数织（Nonogram）不同，数墙的核心是**构建连通的黑色区域**。

---

## 二、官方规则（Nikoli）

### 2.1 基本规则

| 规则 | 说明 |
|------|------|
| **规则1** | 标有数字的方格是**白色**的 |
| **规则2** | 所有**黑色**的方格必须通过边相连形成一个整体（数墙） |
| **规则3** | 任何一个 **2×2** 的区域内不能全是黑色的方格 |
| **规则4** | 每一块白色方格通过边相连形成的区域中有**且仅有**一个标有数字的方格，且这个数字等于这个区域的大小 |

### 2.2 术语说明

| 术语 | 定义 |
|------|------|
| **白色房间** | 连通的白格组成的区域，每个房间有且仅有一个数字格 |
| **数墙** | 所有黑色格子连通形成的区域 |
| **数字** | 表示该格所属白色房间的格子数（不含数字格本身） |

## 三、完整示例

### 3.1 题目解读

```
5 . . . .      ← 数字5表示：这个白色房间有5个格子
2 . . . .      ← 数字2表示：这个白色房间有2个格子
3 5 2 . 5      ← 三个数字分别是3、5、2、5
. . . . 1      ← 数字1表示：这个白色房间只有1个格子
. . . . .      ← 待玩家填充
```

### 3.2 正确解法

```
■ ■ 5 ■ ■      ← ■ = 黑色（数墙的一部分）
■ ■ ■ ■ ■
3 ■ 2 ■ 5      ← 白色房间：数字3所在房间有3格
■ ■ ■ ■ ■      ← 数字2所在房间有2格
■ ■ ■ ■ 1      ← 数字5所在房间有5格，数字1所在房间只有1格
```

## 五、生成器算法说明

### 5.1 算法步骤

```
1. 创建空的5x5网格（全部白色）
2. 随机选择一些格子变成黑色
3. 检查：
   - 黑色格子是否连通？
   - 有没有2x2全黑区域？
   - 每个白色区域是否>=2格？
4. 如果检查通过，记录解
5. 在每个白色区域中随机选一个格子，填入区域大小作为数字
6. 输出题目（数字格+空格）
```

### 5.2 验证函数

```javascript
function verifySolution(grid, numbers) {
  // 1. 数字格必须是白色
  for (let r = 0; r < size; r++) {
    for (let c = 0; c < size; c++) {
      if (numbers[r][c] > 0 && grid[r][c] !== CELL_WHITE) {
        return false;
      }
    }
  }

  // 2. 检查每个白色区域
  const regions = getWhiteRegions(grid);
  for (const region of regions) {
    let numberCount = 0;
    let numberValue = 0;

    for (const [r, c] of region) {
      if (numbers[r][c] > 0) {
        numberCount++;
        numberValue = numbers[r][c];
      }
    }

    // 每个区域必须有且仅有一个数字，且数字=区域大小
    if (numberCount !== 1) return false;
    if (numberValue !== region.length) return false;
  }

  // 3. 检查黑色连通性
  return isValidSolution(grid);
}
```

## 六、玩家技巧

### 6.1 基本技巧

| 技巧 | 说明 |
|------|------|
| **岛屿法** | 数字1只能是单独的1格，周围必须是黑色 |
| **角落法** | 角落的黑格通常会延伸 |
| **排除法** | 如果一个空格周围数字已满足，它必须是黑色 |

### 6.2 高级技巧

| 技巧 | 说明 |
|------|------|
| **连通性保证** | 黑色必须形成单一连通区域 |
| **2×2禁止** | 任何2×2区域不能全黑 |
| **房间大小限制** | 数字N的房间最多有N个白格相邻 |

## 七、难度设计

| 难度 | 网格 | 黑格比例 | 特点 |
|------|------|----------|------|
| 简单 | 5×5 | 15-25% | 数字较多，提示充分 |
| 中等 | 7×7 | 20-30% | 数字较少，需要推理 |
| 困难 | 10×10 | 25-35% | 数字少，陷阱多 |

## 八、常见错误

### 8.1 玩家常犯的错误

| 错误 | 说明 |
|------|------|
| 以为数字格可以是黑色 | ❌ 数字格一定是白色 |
| 以为数字=周围黑格数 | ❌ 数字=所属白色房间的总格子数 |
| 忘记检查2×2规则 | ❌ 任何2×2区域不能全黑 |

### 8.2 示例：错误理解

```
错误理解：
  数字5 = 这个房间周围有5个黑格

正确理解：
  数字5 = 这个白色房间共有5个格子（包括数字格本身）
```

## 十、总结

数墙的核心思维：
1. **黑色是主体** - 要构建连通的数墙
2. **白色是房间** - 每个房间有且仅有一个数字
3. **数字是提示** - 告诉你每个房间有多大
4. **2×2禁止** - 防止形成孤立的黑格
