# 代码规范与审查清单

> 为 FreeTools 团队制定的代码规范，确保代码质量一致、可维护、可协作。

## 一、文件命名规范

### 1.1 页面文件

```
packages/<包名>/pages/<页面名>/
├── <页面名>.js     ← 小写，单词用连字符
├── <页面名>.json
├── <页面名>.wxml
└── <页面名>.wxss
```

**示例**：
```
packages/math/pages/random-selector/random-selector.js   ✓
packages/math/pages/randomSelector/randomSelector.js    ✗
packages/math/pages/RandomSelector/RandomSelector.js  ✗
```

### 1.2 组件文件

```
components/<组件名>/
├── <组件名>.js
├── <组件名>.json
├── <组件名>.wxml
└── <组件名>.wxss
```

## 二、JavaScript 规范

### 2.1 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 常量 | 全大写下划线分隔 | `CACHE_KEY_INDEX` |
| 全局变量 | `_` 前缀 | `_lastPinyin` |
| 普通变量 | 小驼峰 | `battleChain` |
| 函数名 | 小驼峰，动宾结构 | `onBattleConfirm` |
| 私有方法 | `_` 前缀 | `_aiMove` |
| 事件处理 | `on` 前缀 | `onTap` |

### 2.2 Page 数据结构

```javascript
Page({
  data: {
    // ===== 加载状态 =====
    loading: true,
    loadError: '',

    // ===== 用户输入 =====
    inputValue: '',

    // ===== 展示结果 =====
    resultList: [],

    // ===== 业务状态 =====
    currentTab: 0,
  },

  onLoad() {
    this._initData();
  },
});
```

### 2.3 命名空间前缀

| 前缀 | 用途 | 示例 |
|------|------|------|
| `_` | 私有变量/方法 | `_timer`, `_fetchData` |
| `on` | 事件处理 | `onTap`, `onInput` |
| `handle` | 业务处理（非事件） | `handleResult` |
| `$` | DOM/节点引用 | `$canvas` |

### 2.4 箭头函数使用

**使用箭头函数的场景**：
- 回调函数（map/filter/forEach）
- Promise then/catch

**使用普通函数的场景**：
- Page/Component 定义的方法
- 事件处理函数
- 需要 `this` 引用的方法

```javascript
// ✓ 正确
this.data.list.map(item => item.name);
wx.request({ success: res => this._handleRes(res) });

// ✗ 错误：事件处理用箭头函数
onTap: (e) => { this.setData({...}) }  // this 丢失
```

### 2.5 异步处理

```javascript
// ✓ 推荐：Promise + async/await
async _loadData() {
  try {
    const res = await this._fetch(url);
    this._processData(res);
  } catch (err) {
    this._handleError(err);
  }
}

// ✗ 不推荐：回调嵌套
wx.request({
  success: res => {
    wx.request({ success: res2 => { ... } });
  }
});
```

### 2.6 避免的问题代码

```javascript
// ✗ 全局变量污染
count = 0;
tempData = [];

// ✓ 使用 this.data 或私有变量
this.data.count = 0;
this._tempData = [];

// ✗ 硬编码魔法数字
for (let i = 0; i < 60; i++)

// ✓ 常量命名
const MAX_TIMER = 60;
for (let i = 0; i < MAX_TIMER; i++)

// ✗ 直接修改 data（小程序不支持）
this.data.xxx = 123;

// ✓ 通过 setData 修改
this.setData({ xxx: 123 });
```

## 三、WXML 规范

### 3.1 标签使用

| 场景 | 推荐标签 |
|------|----------|
| 容器区块 | `<view>` |
| 文本内容 | `<text>` |
| 图片 | `<image>` |
| 列表渲染 | `<block wx:for>` |
| 条件渲染 | `<block wx:if>` |

### 3.2 属性顺序

```html
<!-- 组件标签顺序：类型 → 绑定 → 数据 → 样式类 → 内联样式 -->
<view
  class="container"
  bindtap="onTap"
  data-id="{{item.id}}"
  style="{{active ? 'opacity:1' : ''}}"
>
  ...
</view>
```

### 3.3 避免内联样式

❌ **不推荐**：
```html
<view style="padding: 20rpx; background: #fff;">
```

✅ **推荐**：
```html
<view class="card">
```
```css
.card {
  padding: 20rpx;
  background: #fff;
}
```

**例外**：动态计算的样式必须用内联：
```html
<view style="width: {{width}}rpx;">
```

## 四、WXSS 规范

### 4.1 选择器优先级

| 选择器 | 优先级 | 说明 |
|--------|--------|------|
| `!important` | 最高 | 尽量避免使用 |
| 内联 style | 高 | 不推荐 |
| class | 中 | 主要使用 |
| id | 中高 | 谨慎使用 |
| 标签选择器 | 低 | 谨慎使用 |
| 通配符 `*` | 最低 | 禁止使用 |

### 4.2 样式组织

```css
/* ===== 变量 ===== */
page {
  --primary: #3498db;
  --text: #333;
}

/* ===== 布局 ===== */
.container { }
.flex-row { }

/* ===== 卡片 ===== */
.card { }

/* ===== 表单 ===== */
.form-row { }
.form-label { }
.form-input { }

/* ===== 按钮 ===== */
.btn { }
.btn-primary { }
.btn:active { opacity: 0.85; }

/* ===== 特殊 ===== */
.tag { }
.result { }
```

### 4.3 rpx vs px

| 场景 | 推荐单位 |
|------|----------|
| 页面整体布局 | rpx |
| 字体大小 | rpx |
| 小间距（1-4rpx） | rpx |
| Canvas 尺寸 | px |
