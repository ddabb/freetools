# 工具添加规范

本文档定义新增工具时的配置规范，违反规范可能导致审核被拒或构建失败。

---

## 1. 工具配置字段说明

在 `config/tools.js` 的 `tools` 数组中添加对象，所有字段均为必填：

```js
{
  id: 'tool-id',
  name: '工具名称',
  icon: '🔧',
  color: 'blue',
  url: '/packages/xxx/pages/xxx/xxx',
  categories: ['分类名'],
  keywords: ['关键词1', '关键词2'],
  description: '一句话功能描述'
}
```

---

## 2. 字段规范

### 2.1 id（工具唯一标识）

| 规则 | 说明 |
|------|------|
| 格式 | 英文小写 + 连字符 `-` |
| 示例 | `length-converter`、`bobing`、`regex-tester` |
| 禁止 | 中文、大写、驼峰、下划线 `_`、特殊字符 |

### 2.2 name（显示名称）

| 规则 | 说明 |
|------|------|
| 长度 | 建议 4~8 个中文字符 |
| 示例 | `长度换算`、`进制转换`、`中秋博饼` |
| 注意 | 审核会扫名称，避免夸大词（"最强""第一"等）|

### 2.3 icon（图标）

| 规则 | 说明 |
|------|------|
| 格式 | 必须是 Emoji（`🎲` `📏` `⚖️`）|
| 来源 | [EmojiAll](https://www.emojiall.com/) 或 [Emoojipedia](https://emojipedia.org/) |
| 禁止 | 图片路径、图片URL、图标组件 |

### 2.4 color（卡片主题色）

| 规则 | 说明 |
|------|------|
| 可选值 | `red` `orange` `yellow` `green` `cyan` `blue` `purple` `pink` `grey` `indigo` `teal` `slate` |
| 示例 | `color: 'blue'` |

### 2.5 url（页面路径）

| 规则 | 说明 |
|------|------|
| 格式 | 相对路径，不含扩展名 |
| 示例 | `/packages/math/pages/sudoku-solver/sudoku-solver` |
| 注意 | 路径末尾不能有 `/` |

### 2.6 categories（所属分类）

| 规则 | 说明 |
|------|------|
| 格式 | 数组，支持多分类 |
| 示例 | `['生活工具']` 或 `['学习工具', '转换工具']` |
| 可选值 | 必须在 `config/categories.js` 中已定义 |

**当前分类列表（优先级顺序）：**
1. 学习工具 2. 生活工具 3. 数学工具 4. 转换工具 5. 财务工具 6. 健康工具 7. 开发工具 8. 安全工具 9. 文本工具 10. 设计工具 11. 更多工具

### 2.7 keywords（搜索关键词）

| 规则 | 说明 |
|------|------|
| 格式 | 字符串数组，每个关键词用单引号包裹 |
| 示例 | `['长度', '单位', '换算', '转换']` |
| 禁止 | 斜杠 `/`（build.js 会报错）、特殊字符、纯英文大写 |

### 2.8 description（功能描述）

| 规则 | 说明 |
|------|------|
| 长度 | 建议 15~30 个中文字符 |
| 内容 | 一句话说明工具用途 |
| 禁止 | 夸大词（最强/第一/全网唯一等）、审核敏感词 |

## 3. 页面文件规范

### 3.1 目录结构

```
packages/<包名>/pages/<页面名>/
  <页面名>.js      # 逻辑
  <页面名>.json    # 配置
  <页面名>.wxml    # 结构
  <页面名>.wxss    # 样式
```

### 3.2 页面 js 模板

```js
const utils = require('../../../../utils/index');

Page({
  data: {
    // 页面状态
  },

  onLoad() {
    wx.setNavigationBarTitle({ title: '页面标题' });
  },

  // 方法...
});
```

### 3.3 页面 json 模板

```json
{
  "usingComponents": {},
  "navigationBarTitleText": "页面标题"
}
```

---

## 4. app.json 分包注册

在 `app.json` 的 `subPackages` 中添加新包：

```json
{
  "root": "packages/bobing",
  "name": "bobing",
  "pages": [
    "pages/bobing/bobing/bobing"
  ]
}
```

---

## 5. CDN 数据规范

### 5.1 文件存放

数据文件放在 `docs/data/<功能名>/` 目录下。

### 5.2 CDN 路径格式

```
https://cdn.jsdelivr.net/gh/ddabb/FreeToolsPuzzle@main/docs/data/<功能>/<文件>.json
```

### 5.3 JSON 格式示例

```json
{
  "changelog": [
    {
      "version": "1.0.0",
      "date": "2026-01-01",
      "type": "feature",
      "title": "功能标题",
      "changes": ["变更点1", "变更点2"]
    }
  ]
}
```

## 6. 审核敏感词清单

以下词汇在工具名、描述、keywords 中**禁止使用**：

| 类别 | 禁止词 |
|------|--------|
| 游戏类 | 游戏、娱乐、博彩、赌博、奖品 |
| 金融类 | 理财（未经许可）、投资（未经许可）、彩票 |
| 社交类 | 聊天、交友、相亲 |
| 内容类 | 新闻（未经许可）、出版 |
| 资质类 | 医疗（需资质）、教育（需资质）、保险 |

建议用"文化""习俗""传统""生活"等中性词替代。

---

## 7. 新增工具检查清单

> 新增工具时逐项确认：

- [x] `config/tools.js` 已添加配置块
- [x] `id` 使用小写连字符格式
- [x] `icon` 是 Emoji
- [x] `color` 是有效颜色值
- [x] `url` 路径正确且不含扩展名
- [x] `categories` 在 categories.js 中已定义
- [x] `keywords` 不含 `/` 或其他特殊字符
- [x] `description` 无夸大词、无审核敏感词
- [x] `app.json` subPackages 已注册分包
- [x] 页面 4 个文件（js/json/wxml/wxss）齐全
- [x] 页面 js 中有 `onLoad` + `wx.setNavigationBarTitle`
- [x] 如有 CDN 数据，已上传到 GitHub 并测试可访问
- [x] 自测：搜索工具名能搜到、点击能正常进入页面
