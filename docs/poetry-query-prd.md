# 诗词查询工具 PRD

## 一、产品概述

### 1.1 产品定位
为用户提供古诗词查询、鉴赏、收藏的轻量级工具，无需后端服务，纯前端实现。

### 1.2 目标用户
- 学生（古诗词学习、背诵）
- 文学爱好者（诗词鉴赏、收藏）
- 普通用户（查询名句、引用）

### 1.3 核心价值
- **离线可用**：无需网络，本地数据
- **快速查询**：按诗人、诗名、名句搜索
- **鉴赏收藏**：注释、译文、赏析

## 三、数据设计

### 3.3 数据转换后的结构

为小程序优化，转换后的数据结构：

**诗词数据**（`poems/[朝代]-[诗人].js`）：
```javascript
export const poems = [
  {
    id: "tang-libai-jingyesi",
    title: "静夜思",
    author: "李白",
    dynasty: "唐",
    form: "五绝",
    content: "床前明月光，疑是地上霜。举头望明月，低头思故乡。",
    tags: ["思乡", "月亮", "五绝"],
    annotation: [  // 需补充
      { word: "床", meaning: "这里指井栏" }
    ],
    translation: "明亮的月光洒在床前...",  // 需补充
    appreciation: "这首诗写的是在寂静的月夜思念家乡的感受...",  // 需补充
    background: "李白于公元726年客居扬州时所作..."  // 需补充
  }
]
```

**诗人数据**（`poets.js`）：
```javascript
export const poets = [
  {
    id: "tang-libai",
    name: "李白",
    dynasty: "唐",
    birth: "701年",
    death: "762年",
    bio: "字太白，号青莲居士，又号\"谪仙人\"...",
    style: "豪放飘逸，想象丰富...",
    poemCount: 1180,
    masterpieces: ["将进酒", "蜀道难", "静夜思"]
  }
]
```

### 3.4 数据处理要点

**原始数据优势**：
- ✅ 朝代分类完整（12个朝代）
- ✅ 诗人信息丰富（Name, Description, Dynasty）
- ✅ 诗词内容完整（按行分割存储）
- ✅ 体裁标签（Form: 五绝、七绝、五律、七律等）

**需要补充的字段**：
- ❌ 注释（annotation）
- ❌ 译文（translation）
- ❌ 赏析（appreciation）
- ❌ 创作背景（background）
- ❌ 题材标签（tags: 山水、田园、边塞等）

**数据清洗工作**：
1. 合并 Contents 数组为完整诗句字符串
2. 生成唯一 ID（如 `tang-libai-jingyesi`）
3. 提取体裁信息（Form 字段）
4. 补充缺失的注释、译文、赏析

### 3.5 数据量预估
- 原始数据：约 50MB（JSON）
- 压缩后：约 10MB（gzip）
- 小程序包限制：20MB（主包）+ 40MB（分包）

## 四、技术实现

### 4.1 技术栈
- **前端**：微信小程序原生框架
- **数据**：本地 JSON + 分片加载
- **存储**：wx.setStorageSync（收藏）

### 4.2 目录结构

```
packages/text/pages/poetry-query/
├── poetry-query.js          // 主逻辑
├── poetry-query.wxml        // 页面结构
├── poetry-query.wxss        // 样式
├── poetry-query.json        // 配置
├── components/
│   ├── poem-card/           // 诗词卡片组件
│   ├── poet-info/           // 诗人信息组件
│   └── search-bar/          // 搜索栏组件
└── data/
    ├── poems/               // 诗词数据（按朝代分片）
    │   ├── tang.js          // 唐诗（约15000首）
    │   ├── song.js          // 宋词（约20000首）
    │   └── ...
    ├── poets.js             // 诗人数据
    └── search-index.js      // 搜索索引
```

### 4.3 性能优化
- **数据分片**：按朝代分片，按需加载
- **搜索索引**：预处理搜索索引，提升查询速度
- **虚拟列表**：长列表使用虚拟滚动
- **懒加载**：诗人详情、诗词详情按需加载

### 4.4 离线支持
- 全部数据内置，无需网络请求
- 首屏数据优先加载（唐诗、宋词）

```
└── server.js          # 本地服务器
```

### 8.3 数据格式特点
- ✅ 每个诗人一个 JSON 文件
- ✅ 包含诗人简介（Description）
- ✅ 包含朝代信息（Dynasty）
- ✅ 包含体裁信息（Form: 五绝、七绝、五律、七律等）
- ✅ 诗词内容按行存储（Contents 数组）

### 8.4 数据处理策略
1. **保留原文结构**：Contents 数组转为完整字符串
2. **生成唯一ID**：`[朝代拼音]-[诗人拼音]-[诗名拼音]`
3. **补充字段占位**：annotation/translation/appreciation 暂为空数组/空字符串
4. **构建搜索索引**：诗人名、诗名、名句分别建立索引

## 九、风险与对策

| 风险 | 对策 |
|------|------|
| 数据量大影响性能 | 分片加载 + 虚拟列表 + 懒加载 |
| 部分诗词缺注释译文 | 标注"暂无"，后续补充 |
| 搜索速度慢 | 预处理索引 + 按朝代分片索引 |
| 存储空间限制 | 数据压缩 + 分包加载 |
| 原始数据格式不一致 | 编写数据清洗脚本 |
